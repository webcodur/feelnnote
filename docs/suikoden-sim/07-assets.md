# 07. 에셋 명세

## 개요

모든 비주얼 에셋은 **픽셀 아트(도트)** 스타일로 통일한다.
해상도 기준: 게임 화면 960×640px (모바일 대응 시 2배 스케일).

---

## A. 캐릭터 에셋

### A-1. 초상화 (필수)

- **용도**: UI 상단, 인물 정보 창, 영입 화면, 전투 HUD
- **규격**: 64×64px
- **수량**: 동적 (현재 ~148, 계속 증가)
- **스타일**: 16색 팔레트, 흉상, 정면 또는 3/4 뷰

**제작 전략 (2가지 중 택 1)**:

| 방식 | 장점 | 단점 | 작업량 |
|------|------|------|--------|
| **A. 개별 제작** | 개성 극대화 | 인물 추가 시마다 제작 필요 | 148+ 장 |
| **B. 템플릿 생성** | 동적 대응 가능, 자동화 | 개성 약함 | 병과 6종 × 성별 2 × 피부톤 3 = 36장 |

> **권장: B안 (템플릿)** — DB에서 동적 로딩하는 구조상 자동 생성이 필수.
> `profession` + `gender` + `nationality(피부톤)` 조합으로 자동 선택.
> 추후 주요 인물(SS/S급)만 개별 초상화 추가.

#### 템플릿 목록 (36장)

```
병과 6종 × 성별 2종 × 피부톤 3종 = 36장

병과: 장수, 책사, 장인, 관료, 예인, 유격
성별: 남(true), 여(false)  // null은 남성 기본
피부톤: 밝음(유럽), 중간(아시아/중동), 어두움(아프리카/남아시아)
  - nationality 기반 자동 매핑
```

### A-2. 맵 스프라이트

- **용도**: 전략 맵에서 캐릭터 이동
- **규격**: 16×16px (맵 위 아이콘)
- **수량**: 병과 6종 × 세력 색상 8종 = 48장
- **애니메이션**: 없음 (정적 아이콘)

### A-3. 전투 스프라이트

- **용도**: 전투 씬에서 캐릭터 행동 표현
- **규격**: 32×48px
- **수량**: 병과 6종 × 상태 4종 = 24장
- **상태**: 대기, 공격, 피격, 사망

```
총 전투 스프라이트: 24장
  장수: 대기/검격/피격/쓰러짐
  책사: 대기/부채질/피격/쓰러짐
  장인: 대기/망치질/피격/쓰러짐
  관료: 대기/명령/피격/쓰러짐
  예인: 대기/연주/피격/쓰러짐
  유격: 대기/투척/피격/쓰러짐
```

---

## B. 맵 에셋

### B-1. 전략 맵 (세계 지도)

- **용도**: 거점 간 이동, 세력 표시
- **규격**: 960×640px (1장)
- **스타일**: 고지도풍 픽셀 아트, 양피지 느낌
- **수량**: 1장

### B-2. 전장 타일셋

- **용도**: 전투 맵 구성
- **규격**: 32×32px per 타일
- **수량**: 12종

```
타일 목록 (12종):
  평지, 숲, 산, 강(가로), 강(세로), 강(합류),
  사막, 설원, 해안, 바다,
  성벽, 성문
```

### B-3. 건물 스프라이트

- **용도**: 거점 경영 화면, 전장 배치
- **규격**: 32×48px
- **수량**: 15종

```
건물 목록 (15종):
  농장, 시장, 교역소, 벌목장, 광산,
  병영, 연병장, 성벽, 무기고,
  도서관, 학당, 사원, 극장,
  첩보부, 외교관저
```
